## Executive Summary

This document outlines a comprehensive strategy for integrating AI-generated images across the Look A Like Solutions website. The approach prioritizes:

- **Conceptual/Illustrative Imagery Only** - No fake people, fake awards, fake certificates, or deceptive client photos
- **Conversion-Focused Visuals** - Each image supports meaning, trust, service clarity, or conversion
- **Premium Digital Marketing Agency Aesthetic** - Clean workspaces, analytics dashboards, search/ad/social motifs, realistic but not deceptive
- **SEO Optimization** - Meaningful alt text, lazy loading, stable aspect ratios, responsive design
- **Performance** - Responsive image wrappers, width/height/aspect-ratio, lazy loading below fold, no layout shift

## Visual Direction Guidelines

### Brand Context
- **Location:** Bengaluru, India (SMB-focused digital marketing agency)
- **Audience:** Small-to-medium businesses seeking digital growth
- **Tone:** Professional, trustworthy, innovative, results-driven
- **Aesthetic:** Modern, clean, analytics-forward, no cheesy stock poses

### Approved Image Categories
✅ **Approved:**
- Service explainer scenes (SEO process, ad setup, social media strategy)
- Process visuals and workflow diagrams
- Analytics dashboards and data visualizations
- Search/ad/social interface mockups
- Lead magnet covers (conceptual, not deceptive)
- Blog hero backgrounds (abstract, conceptual)
- Consultation/strategy session visuals
- Team collaboration scenes (generic, not specific people)
- Workspace environments (clean, professional)

❌ **NOT Approved:**
- Fake people/team members
- Fake client office photos
- Fake awards/certificates
- Fake media logos or press mentions
- Fake client testimonials or screenshots
- Deceptive "before/after" screenshots

## Image Specifications & Technical Requirements

### Responsive Image Implementation
```
- Desktop: Full width, max-width: 1600px
- Tablet: 90% width with padding
- Mobile: 100% width with padding
- Aspect Ratio: Maintained via CSS aspect-ratio property
- Loading: lazy for below-fold, eager for above-fold
```

### Image Optimization
- **Format:** WebP with PNG fallback
- **Sizes:** Generate at 1x and 2x resolution
- **Compression:** Optimize for web (< 200KB per image)
- **Alt Text:** Descriptive, SEO-friendly, 100-150 characters
- **Lazy Loading:** Native HTML lazy loading for below-fold images
- **No Layout Shift:** Always include width/height or aspect-ratio

### Naming Convention
```
Format: [page]-[section]-[type]-[priority].webp
Examples:
- homepage-hero-dashboard-high.webp
- seo-service-process-medium.webp
- blog-card-fallback-medium.webp
- lead-magnet-seo-checklist-medium.webp
```

## SEO & Accessibility Considerations

### Alt Text Strategy
- **Descriptive:** Clearly describe image content and purpose
- **Keyword-Inclusive:** Include relevant keywords naturally (e.g., "SEO optimization process workflow")
- **Length:** 100-150 characters for optimal readability
- **Avoid:** Keyword stuffing, "image of", "picture of"

### Open Graph Image
- **Recommendation:** Use homepage hero dashboard image as default OG image
- **Size:** 1200x630px
- **Format:** WebP with PNG fallback
- **Purpose:** Improve social sharing appearance

### Structured Data
- Consider adding ImageObject schema markup for key images
- Include image URL, alt text, and description in schema
